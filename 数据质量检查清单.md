## 一、案例数据质量检查

### 1. 必填字段检查
每条案例必须包含以下字段，缺一不可：

| 字段 | 检查项 | 通过标准 |
|-----|-------|---------|
| 用户昵称 | 非空 | ✅ 有具体称呼 |
| 城市 | 非空 | ✅ 有城市名称 |
| 用户画像 | 非空 | ✅ 描述用户身份和时间 |
| 平台 | 非空 | ✅ 平台名称正确 |
| 月收入 | 数字 | ✅ 合理范围（任务型300-5000，配送型2000-8000）|
| 每天投入时间 | 非空 | ✅ 有具体时间描述 |
| 经历详情 | 100字以上 | ✅ 有具体内容，不空泛 |
| 遇到困难 | 非空 | ✅ 有具体困难描述 |
| 给新人建议 | 50字以上 | ✅ 有具体可操作建议 |
| 数据来源 | 非空 | ✅ 有来源链接或说明 |

### 2. 内容质量检查

| 检查项 | 说明 | 通过标准 |
|-------|------|---------|
| 真实性 | 是否来自真实用户分享 | ✅ 有来源佐证 |
| 合理性 | 收入数据是否合理 | ✅ 不夸张（不高估也不低估）|
| 细节性 | 内容是否有细节 | ✅ 有具体数字、时间、任务类型 |
| 多样性 | 是否覆盖不同情况 | ✅ 有不同收入、时间、难度 |
| 教育性 | 是否有价值 | ✅ 困难和建议对新人有帮助 |

### 3. 数据分布检查

| 检查项 | 目标分布 | 检查方法 |
|-------|---------|---------|
| 平台分布 | 覆盖主要平台 | 统计各平台案例数量 |
| 收入分布 | 有高有低 | 统计收入分布，避免全部集中在某个区间 |
| 时间分布 | 不同时间段 | 检查开始时间是否分散 |
| 城市分布 | 覆盖不同城市 | 检查是否有地域集中问题 |

### 4. 问题案例处理

| 问题类型 | 处理方式 |
|---------|---------|
| 字段缺失 | 标记，尝试补充，无法补充则剔除 |
| 收入异常 | 检查来源，确认后保留，否则剔除 |
| 内容空泛 | 标记，尝试补充细节 |
| 来源不明 | 降级可信度，或剔除 |
| 重复案例 | 去重，保留最详细的版本 |

## 二、失败/被骗案例质量检查

### 1. 必填字段检查

| 字段 | 检查项 | 通过标准 |
|-----|-------|---------|
| 用户昵称 | 非空 | ✅ |
| 平台/项目 | 非空 | ✅ 骗局名称或平台名称 |
| 损失金额 | 数字 | ✅ |
| 经历详情 | 100字以上 | ✅ 有被骗过程描述 |
| 被骗手法 | 非空 | ✅ 具体描述如何被骗 |
| 教训总结 | 非空 | ✅ 有反思 |
| 警示建议 | 非空 | ✅ 对他人有警示作用 |
| 数据来源 | 非空 | ✅ 优先选择有警方/媒体背书的 |

### 2. 内容质量检查

| 检查项 | 说明 | 通过标准 |
|-------|------|---------|
| 真实性 | 是否有警方通报或媒体报道 | ✅ 优先选择有背书的案例 |
| 教育性 | 是否有警示意义 | ✅ 能帮助他人识别类似骗局 |
| 多样性 | 是否覆盖不同骗局类型 | ✅ 刷单、兼职诈骗、投资诈骗等 |

## 三、平台数据质量检查

### 1. 基本信息检查

| 字段 | 检查项 | 通过标准 |
|-----|-------|---------|
| 平台名称 | 非空 | ✅ |
| 官网 | 可访问 | ✅ 链接有效 |
| 运营主体 | 非空 | ✅ 有公司名称 |
| 运营状态 | 准确 | ✅ 实时核实 |
| 收入范围 | 合理 | ✅ 有数据支撑 |

### 2. 状态核实

| 平台状态 | 核实方法 |
|---------|---------|
| 正常运营 | 官网可访问、APP在商店、近期有用户分享 |
| 问题较多 | 黑猫投诉有投诉、知乎/小红书有负面 |
| 已停运 | 官网无法访问、APP下架、有停运公告 |
| 高风险 | 有跑路新闻、大量投诉未处理 |

## 五、质量标准

### 数据可信度等级

| 等级 | 标准 | 示例 |
|-----|------|------|
| A | 官方通报+媒体报道验证 | 公安部通报的诈骗案例 |
| B | 多用户反馈交叉验证 | 知乎+小红书都有用户分享 |
| C | 单来源或小样本 | 只有一个用户分享 |
| D | 无法验证或可疑 | 来源不明、内容可疑 |
| E | 明显虚假 | 收入夸张、内容空泛 |

### 接受标准
- A级、B级：直接入库
- C级：标记后入库
- D级：人工审核后决定
- E级：剔除

## 六、检查清单使用方法

1. 数据收集完成后，运行自动化检查脚本
2. 导出问题案例列表
3. 人工审核问题案例
4. 生成质量报告
5. 根据质量决定是否需要补充收集
