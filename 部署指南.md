## 二、部署到 GitHub Pages

### 步骤1：创建GitHub仓库
1. 登录 https://github.com
2. 点击右上角 "+" → "New repository"
3. 仓库名填写：`fuye`（或其他名称）
4. 选择 "Public"
5. 点击 "Create repository"

### 步骤2：上传文件
**方式A：网页上传（适合新手）**
1. 在仓库页面点击 "uploading an existing file"
2. 将 `fuye-deploy/` 文件夹内所有内容拖入
3. 点击 "Commit changes"

**方式B：Git命令上传**
```bash
git clone https://github.com/你的用户名/fuye.git
cd fuye
# 将fuye-deploy目录下所有文件复制到这里
git add .
git commit -m "初始部署"
git push origin main
```

### 步骤3：开启GitHub Pages
1. 进入仓库 → Settings → Pages
2. Source 选择 "Deploy from a branch"
3. Branch 选择 "main"，文件夹选择 "/ (root)"
4. 点击 "Save"
5. 等待1-2分钟，访问地址为：`https://你的用户名.github.io/fuye/`

## 三、绑定自定义域名（可选）

### 如果你有自己的域名（如 fuyep.top）

1. **GitHub设置**
   - 仓库 → Settings → Pages → Custom domain
   - 填入：`fuyep.top`
   - 勾选 "Enforce HTTPS"

2. **域名DNS设置**
   - 添加CNAME记录指向：`你的用户名.github.io`
   - 或添加A记录指向GitHub Pages IP：
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```

3. **创建CNAME文件**
   - 在仓库根目录创建文件 `CNAME`
   - 内容填写：`fuyep.top`

## 五、后续更新流程

### 更新案例数据
1. 修改 `data/` 目录下的JSON文件
2. 提交到GitHub：
   ```bash
   git add .
   git commit -m "更新案例数据"
   git push
   ```
3. GitHub Pages会自动重新部署

### 更新页面内容
1. 修改对应的HTML文件
2. 提交到GitHub

## 六、常见问题

### Q: 页面显示空白？
A: 检查JS文件路径是否正确，打开浏览器开发者工具查看Console错误

### Q: AI助手不显示？
A: 确认Bot已发布，检查Bot ID是否正确

### Q: 数据加载失败？
A: 检查JSON文件格式是否正确，确保UTF-8编码
